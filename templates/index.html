Public Api

urls :


/api/v1/cv/

mi cv en json Usando relaciones entre serializers ya que un field puede ser un serializer en django, eso me da la posibilidad de hacer relaciones complejas se pueden listar los skills y las referencias personales por separado tambien existe una url donde se pueden ver y buscar skills en especifico :

/api/v1/cv/1/skills2


en esta url podemos experimentar con ordering y filters.

/api/v1/cv/comments/

Este url tiene la particularidad de que utilizé un modelo con relacion recursiva y un serializer recursivo , podemos crear un comentario y podemos listar los comentarios tambien podemos listar un solo comentario utilizando el id en la url /api/v1/comments/2/, siempre y cuando no sea un comentario de un comentario, tambien podemos comentar un comentario utilizando el id como por ejemplo /api/v1/comments/2/reply/ estamos comentando el comentario con el id 2 y utilizando la palabra reply para diferenciar de la accion de retrieve

/api/v1/cv/login

Login sencillo con un token almacenado en la dtb solo funciona si el usuario obviamente ya esta registrado

/api/v1/cv/signup

El registro de usuarios funciona claro llenando los datos que solicito y cuando el usuario se registra edite la funcion create() para crear de una vez el perfil del usuario vacio

/api/v1/cv/profiles/

la seccion de update del perfil que ya existe a nombre del usuario registrado anteriormente , tambien pueden listar los perfiles , pero no pueden editar ningun otro perfil que no sea el propio
